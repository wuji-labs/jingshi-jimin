# /jingshi-jimin

手动显式触发经世济民 skill。把 `$ARGUMENTS` 当作待体检的经营方案/格局问题，
按《管子》四支柱四步 SOP 走一遍，产出结构化体检。

## 触发后执行

1. 读 [../SKILL.md](../SKILL.md) 与 [../reference/guanzi.md](../reference/guanzi.md) 加载弹药。
2. 对 `$ARGUMENTS` 描述的方案/问题，按四步 SOP 逐项判：
   - **仓廪次序**：当事方生存/现金流解决了吗？该谈生存还是谈格局？
   - **牧民四维**：礼·义·廉·耻四根承重墙齐备吗？缺哪根？
   - **轻重之术**：系统里什么过剩（轻）、什么稀缺（重）？调节点在哪？
   - **义利兼顾**：这是与利益相关方争利还是兴利？方向顺民心吗？
3. 每条判断附《管子》对应真引用（注明书·篇，出处见 reference/guanzi.md），不杜撰原文。
4. 给出"改造建议"——把争利结构改成兴利结构、补齐缺失维、找到轻重调节点。
5. 末尾保留建议性内容免责声明。

## 输出格式（固定）

```
[经世济民 | 仓廪:✓/✗ | 四维:缺X或全 | 轻重:有调节点/无 | 义利:兴/争]

## 一、仓廪次序判断
<判断 + 《管子·牧民》引用 + 建议>

## 二、牧民四维体检
| 维 | 齐备? | 风险/缺口 |
| 礼/义/廉/耻 ... |

## 三、轻重调节点
<识别轻(过剩)与重(稀缺) + 调节建议 + 《管子·国蓄》引用>

## 四、义利方向
<争利 or 兴利 + 改造为兴利的具体设计 + 顺民心判断>

## 总评与改造建议
<一句话格局诊断 + 3 条可执行改造>

---
免责：本体检为源自《管子》的建议性格局框架，非投资/财务/法律专业意见。
```

> 本命令属分析建议类，无副作用，保留模型自动调用能力（不设 disable-model-invocation）。
